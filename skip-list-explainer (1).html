The skip list

A sorted linked list with built-in shortcuts. Extra "express lanes" let a search hop over big chunks of the list, turning a slow O(n) walk into an O(log n) glide — without the bookkeeping of a balanced tree.

How it works

Every value lives on the bottom lane, a normal sorted linked list. When a node is inserted, a coin is flipped repeatedly — each heads promotes it one lane higher. So roughly half the nodes reach lane 1, a quarter reach lane 2, and so on.

Searching starts top-left and moves right while the next value is still ≤ the target. When the next value overshoots, it drops down a lane and keeps going. The high lanes skip past most of the list; the bottom lane confirms the exact match.

Why use one

It matches a balanced binary search tree's speed but is far simpler to implement — no rotations or rebalancing, just coin flips. The randomness makes it naturally easy to write concurrent versions, which is why it shows up in systems like Redis sorted sets and many in-memory databases.
